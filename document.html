              │  ArtifactLoader              │

              │                              │

              │  local cache into memory:    │
              │  • Token groups              │

              └──────────────────────────────┘

Key Points

  • The manifest check happens once at app startup.
  • Only changed files are downloaded (hash-based diffing), not everything.
  • Registry data (tokens, styles, components) is preloaded into memory so compilation is instant.
  • If the server is unreachable, the app gracefully falls back to the local cache.

3. Manifest Sync & Caching

Manifest JSON Structure (from backend)

{
  "version": "1.0.0",
  "tokens": {
    "colors": { "path": "http://host:5050/registry/tokens/colors", "hash": "sha256:abc..." },
    "spacing": { "path": "http://host:5050/registry/tokens/spacing", "hash": "sha256:def..." }
  },
  "styles": {
    "buttons": { "path": "http://host:5050/registry/styles/buttons", "hash": "sha256:ghi..." }
  },
  "components": {
    "header": { "path": "http://host:5050/registry/components/header", "hash": "sha256:jkl..." }
  },
  "screens": {
    "login": { "path": "http://host:5050/screens/login", "hash": "sha256:mno..." },
    "dashboard": { "path": "http://host:5050/screens/dashboard", "hash": "sha256:pqr..." }
  }
}

Diff Algorithm

For each category (tokens, styles, components, screens):
  For each entry in server manifest:
    IF entry does not exist locally OR local hash ≠ server hash:
      → Download the file from entry.path
      → Save to LocalStorage
      → Mark as "updated"

Version Bumping

  • The version field changes when registry files (tokens, styles, components) change.
  • When version changes, all cached artifacts become stale and are recompiled on next access.
  • Screen-only changes don't bump the version — only the screen's own hash changes.

Priority Source When Used
1st Local cache Screen was previously downloaded from backend
2nd Bundled assets Screen JSON is shipped inside the APK/IPA
3rd Remote HTTP Fallback fetch from backend server

Render Modes

Mode Description
artifact Compiled artifact with computedStyle per node (primary V3 path)
rawJson Uncompiled JSON, uses V2 runtime resolvers as fallback

5. Compilation Pipeline

The compiler runs client-side and transforms raw screen JSON + registry data into a compiled artifact.

┌─────────────────────────────────────────────────────────────────┐
│                       SduiCompiler.compile()                    │
└────────────────────────────┬────────────────────────────────────┘
                             │
         ┌───────────────────┼───────────────────┐
         ▼                   ▼                   ▼
┌──────────────────┐ ┌──────────────────┐ ┌──────────────────┐
│  TokenCompiler   │ │  StyleCompiler   │ │ ComponentCompiler│
│  .load()         │ │  .load()         │ │  .load()         │
│                  │ │                  │ │                  │
│  Flattens token  │ │  Indexes style   │ │  Indexes comp    │
│  groups into     │ │  definitions by  │ │  definitions by  │
│  dot-path map    │ │  .file.key       │ │  name            │
└──────────────────┘ └──────────────────┘ └──────────────────┘
                             │
                             ▼
              ┌──────────────────────────────┐
              │   Parse raw JSON → SduiRoot  │
              │   (layout tree, apis,        │
              │    actions, inline V2 data)  │
              └──────────────┬───────────────┘
                             │
                             ▼
┌─────────────────────────────────────────────────────────────────┐
│          _compileNode(node, parentStyle) — RECURSIVE            │
│                                                                 │
│   For EACH node in the tree:                                    │
│                                                                 │
│   ① Is type == "component"?                                     │
│      YES → ComponentCompiler.expand(node)                       │
│            Inline the component's subtree definition             │
│            → Re-enter _compileNode with expanded node           │
│                                                                 │
│   ② StyleCompiler.compute():                                    │
│      ┌─────────────────────────────────────────────────────┐    │
│      │  Layer 1: Parent's computed style (inheritance)     │    │
│      │  Layer 2: Style class refs  (.file.key → resolved)  │    │
│      │  Layer 3: Inline style overrides (inlineStyle map)  │    │
│      │                                                     │    │
│      │  Merge order: Layer 1 ← Layer 2 ← Layer 3          │    │
│      │  (Later layers override earlier ones)               │    │
│      │                                                     │    │
│      │  Then: TokenCompiler resolves all #group.key refs   │    │
│      │  e.g. #colors.primary → "#0A84FF"                   │    │

│      │  Output: flat computedStyle map                     │    │
│      └─────────────────────────────────────────────────────┘    │
│                                                                 │
│   ③ Recursively compile children[]                              │
│     (pass current computedStyle as parentStyle)                 │
│                                                                 │
│   ④ Return node.copyWith(                                       │
│        computedStyle: mergedFlatMap,                            │
│        clearStyle: true,     ← remove class refs               │
│        clearInlineStyle: true ← remove inline overrides        │
│      )                                                          │
└─────────────────────────────────────────────────────────────────┘
                             │
                             ▼
┌─────────────────────────────────────────────────────────────────┐
│                      SduiArtifact (Output)                      │
│                                                                 │
│  {                                                              │
│    "artifactVersion": "3.0",                                    │
│    "screen": "login",                                           │
│    "artifactKey": "login_1.0.0",                                │
│    "registryVersion": "1.0.0",                                  │
│    "renderTree": {                                              │
│      "type": "screen",                                          │

│      },                                                         │
│      "children": [ ... nodes with flat computedStyle ... ]      │
│    },                                                           │

Token Resolution Examples

Input (in style) Resolved Output
#colors.primary "#0A84FF"
#spacing.md 16
#FFFFFF "#FFFFFF" (literal hex — not a token, passed through)
{{form.mobile}} "{{form.mobile}}" (runtime binding — not resolved at compile time)

Style Merge Example

Parent computed:    { "color": "#212121", "fontSize": 14 }
Class .heading:     { "fontSize": 26, "fontWeight": "bold" }
Inline override:    { "color": "#FF0000" }

Result computedStyle:
{
  "color": "#FF0000",         ← inline wins over parent
  "fontSize": 26,             ← class wins over parent
  "fontWeight": "bold"        ← from class
}

6. Rendering Pipeline

The renderer takes a compiled artifact and builds Flutter widgets.

┌─────────────────────────────────────────────────────────────────┐
│          SduiScreenWidget (StatefulWidget)                       │
│                                                                 │

│  ┌───────────────────────────────────────────────────────┐      │

│                         │                                       │

│  │  GetBuilder<SduiStateController>                         │   │

│  │        │                                                 │   │
│  │        ▼                                                 │   │

Style Resolution (V3 vs V2)

Step V3 Artifact Mode V2 Raw JSON Fallback
Token resolution Done at compile time Done at runtime by TokenResolver
Style class merging Done at compile time Done at runtime by StyleResolver
Read style for widget node.computedStyle → ComputedStyleParser node.style → StyleResolver.resolve()
Performance Fast (flat map lookup) Slower (resolution + merge per frame)

7. Action Execution Flow

Actions are triggered by user interactions (tap, submit) or lifecycle events.

┌─────────────────────────────────────────────────────────────────┐
│                     User Interaction                             │
│         (tap button, submit form, screen loads, etc.)           │
└────────────────────────────┬────────────────────────────────────┘
                             │
                             ▼
              ┌──────────────────────────────┐
              │  ActionExecutor.run(action)   │
              └──────────────┬───────────────┘
                             │
                    ┌────────┴────────────────────────────────┐
                    │     Check condition (if present)         │
                    │     ExpressionResolver.resolveBool()     │
                    │     false → SKIP action entirely         │
                    └────────┬────────────────────────────────┘
                             │ true (or no condition)
                             ▼
┌─────────────────────────────────────────────────────────────────┐
│                   Switch on action.type                          │
│                                                                 │
│  NAVIGATION                                                     │
│  ─────────────                                                  │
│  "navigate"   → GetX push to named route                       │
│  "pop"        → GetX pop (go back)                              │
│                                                                 │
│  DATA / API                                                     │
│  ─────────────                                                  │
│  "api"        → HTTP call via ApiClient                         │
│                 → Resolve {{form.*}} in body                    │
│                 → onSuccess / onFail → run sub-actions          │
│  "setState"   → Update SduiStateController state key            │
│  "setForm"    → Update SduiStateController form field           │
│  "storeToken" → Save value to secure storage                    │
│                                                                 │
│  UI FEEDBACK                                                    │
│  ─────────────                                                  │
│  "toast"      → Show SnackBar message                           │
│  "popup"      → Show dialog / bottom sheet                      │
│  "launchUrl"  → Open URL in external browser                    │
│                                                                 │
│  FLOW CONTROL                                                   │
│  ─────────────                                                  │
│  "chain"      → Run actions[] array sequentially                │
│  "delay"      → Wait N milliseconds, then continue             │
│  "condition"  → If/else branching (run different actions)       │
│                                                                 │
│  EXTENSIBILITY                                                  │
│  ─────────────                                                  │
│  "custom"     → Call registered handler from CustomActionRegistry│
│  "event"      → Fire named event (inter-component messaging)   │
│  "analytics"  → Fire analytics event to handler                 │
└─────────────────────────────────────────────────────────────────┘

Action Chaining Example (Login Flow)

{
  "event": "onClick",
  "type": "api",
  "target": "sendOtp",
  "onSuccess": {
    "type": "chain",
    "actions": [
      { "type": "setState", "props": { "key": "state.mobile", "value": "{{form.mobile}}" } },
      { "type": "toast", "props": { "message": "OTP sent!" } },
      { "type": "navigate", "target": "/otp" }
    ]
  },
  "onFail": {
    "type": "toast",
    "props": { "message": "Login failed. Try again." }
  }
}

Execution:

  1. API call sendOtp → POST /api/auth/send-otp with { "mobile": "9876543210" }
  2. On success → chain runs sequentially:
    • Save mobile to state
    • Show toast "OTP sent!"
    • Navigate to /otp screen

10. Caching Strategy

Cache Layers

┌─────────────┐       ┌───────────────────┐       ┌──────────────┐
│   Backend    │──────►│  ManifestChecker  │──────►│ LocalStorage │
│   Server     │  GET  │  (hash-based diff)│ save  │              │
│              │       └───────────────────┘       │ Cached:      │
│ /manifest    │                                    │ • manifest   │
│ /registry/*  │                                    │ • tokens/*   │
│ /screens/*   │                                    │ • styles/*   │
└─────────────┘                                    │ • comps/*    │
                                                   │ • artifacts/*│
                                                   └──────┬───────┘
                                                          │
                                   ┌──────────────────────┴──────┐
                                   │                             │
                              Cache HIT                     Cache MISS
                              + version matches             or STALE
                                   │                             │
                                   ▼                             ▼
                            Return instantly            Compile raw JSON
                            (0ms latency)               → Cache artifact
                                                        → Return

What Gets Cached

Item Storage Key Invalidated When
Manifest sdui_manifest Every successful server fetch
Token files registry_tokens_{id} Hash changes in manifest
Style files registry_styles_{id} Hash changes in manifest
Component files registry_components_{id} Hash changes in manifest
Raw screen JSON artifact_{screenId} Hash changes in manifest
Compiled artifacts artifact_{screenId} Registry version changes

Cache Invalidation Rules

  1. Registry changes (tokens/styles/components) → manifest.version bumps → all cached artifacts become stale → recompiled on next access
  2. Screen-only changes → only that screen's hash changes → only that screen is re-downloaded and recompiled
  3. No changes → everything served from cache → zero network calls after startup
